> **版本：v1.0 | 2026-03-12**
>
> **定位**：在个股研究完成后，决定是否买入、买多少、怎么管。**依赖当前持仓状态与宏观环境**，时间敏感。
>
> - **输入**：「个股研究SOP.md」产出的研究结论摘要 + 当月月度快照

> - **黄金/对冲/债券操作** → 查阅「资产配置框架.md」第四章
> - **不确定该看哪个文档？** → 查阅「投资体系总览.md」

## 一、投资哲学层级声明

### 1.1 工具层级（冲突时谁优先）

```
第0层（生存约束）：达利欧 → 资产配置框架.md
  → 股权层上限由宏观象限 × 市场温度动态决定，每月更新快照
  → 任何标的的仓位不能突破当前股权层上限

第1层（核心框架）：巴菲特
  → 个股研究SOP已在第1-4步完成质量判断
  → 本文件基于研究结论摘要做仓位决策

第2层（否决层）：芒格
  → 已在个股研究SOP第4步执行；进入本文件意味着已通过否决门

第3层（仓位控制层）：加尔布雷思
  → 市场温度影响仓位上限和新增资金节奏
  → 不作为个股卖出触发，不改变质量/估值标准
```

**冲突处理**：层级高的框架优先。两个同层工具方向相反时，仓位自动降低50%，标注为"存疑持仓"加强跟踪。

## 二、决策流程（三步）

```
决策第1步：宏观定位
  引用当月月度快照（output/月度快照_YYYY-MM.md）
  → 确认总股权层上限 + 各市场子上限
  → 确认当前实际配比 vs 目标配比，偏差>5%先再平衡
        ↓
决策第2步：行业热度检查
  引用资产配置框架.md「行业热度检测」当月评级
  → 确认目标标的所属行业的热度评级
  → 🔴过热 → 仓位额外折扣50%  |  🟡偏热注意 → 折扣20%  |  🟢合理 → 无折扣  |  🔵低温 → 可适当加大
        ↓
决策第3步：仓位决策
  引入研究结论摘要（来自个股研究SOP）
  → 三维软性评分 × 市场温度修正 × 行业热度折扣 × 治理系数
  → 输出：操作决策单
```

### 决策第2步：行业热度检查

> 引用当月月度快照「行业热度检测」部分结论，不重新计算。

**操作模板**：

```
目标标的：______  所属行业：______  市场：A股/港股/美股

行业热度评级（来自月度快照）：
  评级：□ 🔴过热  □ 🟡偏热注意  □ 🟢合理/正常  □ 🔵低温/被忽视
  评级依据（引用快照结论）：______

行业热度仓位折扣：
  🔴过热     → 目标仓位额外折扣50%
  🟡偏热注意  → 目标仓位额外折扣20%
  🟢合理/正常 → 无折扣
  🔵低温/被忽视→ 可适当加大（在市场子上限范围内）

→ 本标的行业热度折扣系数：___（0.5 / 0.8 / 1.0 / 1.0+）
```

> **注**：若标的行业未在快照中覆盖 → 默认🟢合理，折扣系数=1.0，并在决策单中注明"行业热度未评级"。

### 决策第3步：仓位决策

> 输入：来自「个股研究SOP.md」的研究结论摘要。若研究结论已过期（超过触发复查事件或6个月），回到研究SOP重做后再进行此步。

#### 3.1 三维软性评分

从研究结论摘要中提取：

| 维度 | 评分 | 来源 |
|------|------|------|
| 定性评分 | ✅/⚠️/❌ | 研究结论摘要 |
| 财报评分 | ✅/⚠️/❌ | 研究结论摘要 |
| 估值评分（以DCF区间为准） | ✅/⚠️/❌ | 研究结论摘要 |

**估值评分规则**（若研究结论摘要中未单独列出，按此判断）：

| 当前股价区间 | 估值评分 |
|------------|---------|
| 低于DCF悲观情景 | ✅ |
| DCF悲观～基准之间 | ⚠️ |
| 接近或高于DCF基准情景 | ❌（除非护城河极强） |
| 高于DCF乐观情景 | ❌ |

#### 3.2 组合允许仓位计算

```
基础仓位上限（按三维软性评分）：
  3维全✅ → 可用仓位的20%（重仓上限）
  2维✅ → 可用仓位的10%（小仓位试探）
  1维✅及以下 → 观望，不开仓

× 市场温度修正（来自第1步）：
  极度恐慌 → +20%  |  悲观冷清 → +10%  |  冷静正常 → ±0%  |  乐观偏热 → −10%  |  极度狂热 → −20%

× 行业热度折扣系数（来自第2步）：
  🔴过热 → ×0.5  |  🟡偏热 → ×0.8  |  🟢合理 → ×1.0  |  🔵低温 → ×1.0

× 治理系数（来自研究结论摘要）：
  ___（1.0=无命中；最低项=___）

→ 综合仓位上限 = 基础 × 市场修正 × 行业折扣 × 治理系数
  注：最终仓位不得突破第1步确认的市场子上限
```

> **市场子上限约束**：综合仓位上限与市场子上限取**低值**。例如，计算得15%，但A股子上限≤20%，当前A股已有18%，则本标的最多只能再加2%。

#### 3.3 仓位类型（按林奇分类）

| 林奇分类 | 仓位类型 | 说明 |
|---------|---------|------|
| 稳定增长型 | **主力仓位** | 判断简单，容错高，基本面破坏才卖 |
| 快速增长型 | **卫星仓位** | 小仓位，PEG<1时介入，增速下台阶立即清仓 |
| 困境反转型 | **机会仓位** | 仅限曾经好公司暂时遇难，严格小仓位 |
| 缓慢增长型 | **ETF替代** | 高股息ETF比个股更省心 |
| 周期型⛔ | 不碰 | 研究SOP已在第1步否决，不应到达此步 |

## 四、ETF简化流程

> 宽基/行业ETF没有单一商业模式和财报，跳过「个股研究SOP.md」的第1-4步。但需执行ETF产品结构红线和估值判断。

### ETF决策流程（三步）

```
ETF第1步：宏观定位
  → 与个股决策第1步相同：引用月度快照，确认股权层上限 + 市场子上限

ETF第2步：产品结构红线（硬门槛）
  → 确认"买得到也卖得掉"，结构不引入额外非线性风险

ETF第3步：估值分析
  → 按指数类型选对估值指标，查百分位判断贵贱
```

### ETF产品结构红线（命中任一条 → 直接否决）

| 红线 | 信号 | 动作 |
|------|------|------|
| **杠杆/反向/复杂策略** | 杠杆/反向/期权增强/波动率等结构 | 不作为底仓（仅对冲层使用，且完全理解） |
| **合成复制/对手方风险** | Swap/复杂衍生品/显著展期风险 | 直接否决（不懂不做） |
| **流动性与规模** | 基金规模/日均成交额/价差不满足最低阈值 [来源,日期] | 否决（避免"卖不掉"） |
| **费率与跟踪偏离** | 费率/跟踪误差超出可接受区间 [来源,日期] | 否决 |

### ETF估值指标速查（按指数类型）

| 指数类型 | 主估值指标 | 辅助指标 | 典型指数 |
|---------|-----------|---------|---------|
| 大盘宽基 | PE-TTM百分位 | 席勒PE（美股） | 沪深300、标普500、恒生指数 |
| 中小盘宽基 | PE-TTM百分位 | PB百分位 | 中证500、中证1000 |
| 成长型 | PE中位数百分位 | PS百分位 | 创业板指、科技龙头、纳斯达克100 |
| 红利/高股息 | PE-TTM百分位 | **股息率** | 中证红利、上证红利 |
| 金融行业 | **PB百分位** | 股息率（银行） | 中证银行、证券公司 |
| 消费/医药 | PE-TTM百分位 | — | 中证消费、中证医药 |
| 周期行业 | PE-TTM百分位 | PB百分位 | 中证军工、基建工程 |

> 完整指数评估表见「估值工具手册.md」第二、四章。

### ETF定投与市场温度对照

| 市场温度 | 定投策略 |
|---------|---------|
| 极度恐慌 | 加大定投（平时2倍） |
| 悲观冷清 | 适当增加定投（平时1.5倍） |
| 冷静正常 | 维持正常定投 |
| 乐观偏热 | 减少定投（平时一半），但不停止 |
| 极度狂热 | 暂停定投，考虑适当减仓 |

> **定投本身就是格雷厄姆"安全边际"在指数投资中的最佳实践**——高位自然买少，低位自然买多。

## 五、协作复盘机制

> 每次协作session结束前，AI主动发起或配合用户进行复盘。

| 复盘维度 | 内容 | 谁来做 |
|---------|------|-------|
| **本次做了什么** | 列出本次协作的所有变更（文档/报告/框架） | AI整理 |
| **哪里出了问题** | 错误、遗漏、不够准确的地方 | 双方讨论 |
| **新发现的错误模式** | 是否触发了新的E类错误？是否需要新增自检项？ | AI自查+用户反馈 |
| **迭代记录** | 将变更和教训写入迭代优化池 | AI执行 |
